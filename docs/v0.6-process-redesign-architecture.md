# GreenChemistry.ai v0.6: Process Redesign Architecture

## 1. Goal & Vision
To evolve GC.ai from a chemical substitution tool (v0.5) into a **Process Redesign Partner**. v0.6 moves beyond "Swap A for B" to propose structural changes to the synthesis—reducing steps, optimizing energy, and identifying greener pathways—without overclaiming experimental certainty.

## 2. The "Equivalent Process" Claim
A process redesign is higher risk than a material swap.
- **v0.5 (Material Swap):** Claims "Equivalent Solvent/Reagent performance."
- **v0.6 (Process Redesign):** Claims **"Theoretically Equivalent Outcome."** 
- **UX Constraint:** Redesign proposals must be clearly demarcated from substitutions and require explicit scientist validation (e.g., "This route eliminates Step 3, but may require re-optimizing workup").

## 3. Architecture: Deterministic vs. LLM-Assisted
We maintain a "Safe Core" (deterministic) with a "Creative Layer" (LLM).

### Deterministic Components (Chemistry Service)
- **Mass Balance:** Calculating theoretical Atom Economy (P2) and PMI (P1) for the new route.
- **Condition Scoring:** Temperature/Pressure severity scoring (P6).
- **Baran Ideality Check:** Quantifying "Ideal Synthesis" metrics (r/p ratio, step count).
- **Hazard Delta:** Calculating the net reduction in toxicity/waste mass between routes.

### LLM-Assisted Components
- **Pathway Generation:** Proposing alternative reagents or catalyst systems (e.g., "Use biocatalysis instead of metal catalysis").
- **Step Consolidation:** Suggesting "one-pot" sequences or telescoping steps.
- **Narrative Rationale:** Explaining *why* a redesign is better and citing specific Green Chemistry principles (P8, P9).

## 4. Data Model Extensions

### AnalysisStep (Enhanced)
- `id`: Unique step identifier for tracking across redesigns.
- `isTelescoped`: Boolean, indicates if it can be combined with the next step.
- `routeSegment`: Identifier (e.g., "Core Formation", "Deprotection").

### RedesignProposal (New Entity)
- `type`: "Catalytic Improvement" | "Step Elimination" | "Route Optimization".
- `impactedSteps`: `stepNumber[]`.
- `newProcedure`: Monospace protocol text for the redesigned section.
- `deltaMetrics`:
  - `stepReduction`: number
  - `pmiImprovement`: number
  - `atomEconomyGain`: number
- `experimentalCaveats`: Array of specific technical risks (e.g., "Increased viscosity might hinder stirring").

## 5. Pipeline Evolution (Phase 4b Integration)
Phase 4b moves from "Placeholder" to "Process Redesign Engine":
1. **Input:** Enriched chemicals (Phase 2) + Material recommendations (Phase 4a).
2. **LLM Prompt:** Focus on Principles 1, 2, 6, 7, 8, 9.
   - "Given these material swaps, can we eliminate steps? Can we switch from batch to flow? Can we use a catalyst to lower Temp from 120C to 40C?"
3. **Output:** A set of `RedesignProposal` objects.

## 6. Scoring Changes (v0.6)
- **Composite Score:** The "Grade" now weights **Process Efficiency (P1, P2, P8)** higher than in v0.5.
- **Uncertainty Penalty:** Redesign scores carry a `confidence: "theoretical"` flag to distinguish from PubChem-backed material scores.

## 7. UX Boundaries
- **Visual Separation:** Redesign proposals appear in a separate UI tab or use a "Blueprint" icon to differentiate them from simple chemical "Swap" cards.
- **"The Scientist in the Loop":** Users must "Approve" a redesign before it's integrated into the Finalized Protocol.
- **Citation Requirement:** Every redesign proposal must cite a representative paper or general principle (e.g., "Guided by P9: Catalysis").

## 8. Rollout Plan
1. **v0.6a (Beta):** Implement `RedesignProposal` data structure and Phase 4b prompt engineering.
2. **v0.6b (Alpha Test):** Run redesign logic against 50 known pharmaceutical synthesis routes (e.g., Ibuprofen, Sildenafil) to calibrate "PMI improvement" claims.
3. **v0.6c (Production):** Launch with "Experimental" tags on all redesign features.
